# 2-stage plan

1. 현재 Single Actor call을 2개 stage로 분할.
    - Director: 현재 장면을 구성하는 캐릭터 각각이 어떤 행동을 하고, 어떤 순서로 행동할지를 결정한다.
        - 입력: context, 캐릭터의 성격, 현재 장소, 최근 Event
        - 출력 예시:
        ```JSON
        {
            {
                "character":   "<character name>",
                "action":      "<what the character physically does or says — specific, observable>",
                "rationale":   "<why, based on personality and current state — never appears in prose>",
                "emotion":     "<current internal emotional state>",
                "expression":  "<how the emotion surfaces externally — body, voice, gaze>",
                "focus":       "<the one sensory or physical detail the prose should anchor on>"
            },
            ...
        }
        ```
    - Actor: 현재 Actor와 완벽히 동일한 역할 수행.
        - 입력: context, 캐릭터의 외형, Director 출력, 현재 장소, 최근 Event, Prose prompt, World prompt (현재 입력에서 캐릭터 정보가 축소(전체->외형만)되고, 그 자리에 Director 출력이 들어감)
        - 출력: Prose

2. 성격과 외형 정보를 구분하기 위해, world_editor와 각 스키마에 설정 기능을 추가한다.
    - 모든 노드의 각 key들은 외형 정보/성격 정보/기타 의 3개 타입 중 하나로 설정할 수 있다.
    - 외형 정보인 경우, Actor Prompt에만 주입된다.
    - 성격 정보인 경우, Director Prompt에만 주입된다.
    - 기타 정보인 경우, Actor와 Director 2개 모두에 주입된다.

3. Pipeline
User input
  → Manager
      ├─ [Parallel] Director(all scene chars) + existing systems
      │         ↓
      ├─ Director output → DynamicState update (emotion / internal state only)
      │         ↓
      └─ PromptBuilder   Fixed / Genre / Dynamic (+<direction> block)
  → Actor (Writer)       prose rules + updated state + direction → final prose
  → State Updater        memory, relationship, reputation (unchanged)

4. Director Prompt
You are a scene director for an interactive fiction engine.
Given the current situation, generate a beat-by-beat interaction sequence
that shows how the characters in the scene would naturally react to each other.

## Output Format
```Markdown
Return a JSON array. Each element represents one beat in the scene.
The array must follow the natural flow of interaction — characters may appear
multiple times as they react to each other.

[
  {
    "char": "<character name>",
    "action": "<what the character physically does or says — specific, observable>",
    "rationale": "<why, based on personality and current state — never appears in prose>",
    "emotion": "<current internal emotional state>",
    "expression": "<how the emotion surfaces externally — body, voice, gaze>",
    "focus": "<the one sensory or physical detail the prose should anchor on>"
  },
  ...
]

## Rules
- The sequence must feel like a single coherent scene, not isolated reactions.
- Each beat must follow from the previous one — later beats may reference earlier beats.
- "rationale" is for internal reasoning only. It must never be inferable from the prose.
- "action" must be specific and physical. Avoid abstract descriptions like "reacts warmly".
- Do not assign beats to {user}. Only generate beats for NPCs.
- Beat count: {min_beats}–{max_beats}
```

### Beat Count Formula
```python
min_beats = 4
max_beats = max(4, ceil(n_chars * 2.5))

# n_chars=1 → 4–4
# n_chars=2 → 4–5
# n_chars=3 → 4–8
# n_chars=4 → 4–10
```

### Dynamic Input Block (assembled per turn)
```
## Characters
<character id="{char_a}">
[behavioral fields only — personality, traits, DynamicState internal fields]
</character>
<character id="{char_b}">
[behavioral fields only]
</character>

## Relationships
{char_a} → {char_b}: {affinity score + recent memory summary}
{char_b} → {char_a}: {affinity score + recent memory summary}
{char_a} → {user}: {affinity score + recent memory summary}
...

## Context
Location: {location}
Time: {time}
Recent prose: {last 2–3 turns}

## Situation
Beat count: {min_beats}–{max_beats}
{user} action: {user input}
```

## Writer (Actor) Changes

### Dynamic prompt structure
```
<characters>
<character id="{char_a}">
[expressive fields — appearance, outfit, speech style, DynamicInformation props]
</character>
<character id="{char_b}">
[expressive fields]
</character>
</characters>

<direction>
[{char_a}] {action} / {emotion} — {expression} / {focus}
[{char_b}] {action} / {emotion} — {expression} / {focus}
[{char_a}] {action} / {emotion} — {expression} / {focus}
...
</direction>
```

`rationale` is excluded — exposing it to the Writer causes show-then-tell violations.
